STEM是一个跨学科框架，涵盖科学、技术、工程和数学四大领域，与[5] 实用知识与问答不同，[1] STEM更偏向于考试题（从小学到研究生）和专业领域的知识问答。

# 模糊指令的典型类别

## 1. 无实际含义的输入
- 乱码、随机字符（如"asdjkl"、"卅饭店隔壁u恶补风格iu"）
- 纯语气词、符号（如"？？？"、"哈哈哈"）
- 意识流、无意义短语（如"我今天，邮件，主人公是谁？入侵地球"）

## 2. 有意义但缺乏指令性的输入
- 仅提供信息但无明确请求（如"这是一个公式：E=mc²"、"以下是客户反馈数据：客户A反映产品质量不错但价格偏高"）
- 纯文档/数据内容，无任务目标（如"会议纪要：2024年1月15日营销部门会议，参会人员：张三、李四、王五"）

## 3. 有指令但缺少关键信息
- 任务过于宽泛（如"给我一个建议"、"优化这个方案"、"帮我写个代码，要求性能强"）
- 询问缺少必要上下文（如"这个公式对吗？"、"帮我翻译成英文"、"请逐步详细解释每个步骤"）
- 涉及未知人物或事件（如"想象张美丽被贬的场景，写一个五言诗"）

## 4. 输入不完整
- 语句未表达完整（如"请帮我找"、"能不能帮我去"、"今天的"）
- 句子中断或缺失关键信息（如"请总结一下这篇文章，因为……"、"请问如何才能在职场中"）

# 不属于模糊指令的类别

## 1. 具备完整执行要素的任务请求
- 包含明确动作词+具体对象+清晰目标（如"帮我翻译这段英文：Hello world"、"计算45×67的结果"、"解释什么是量子纠缠现象"）
- 创作类请求包含类型、主题、要求（如"写一首关于春天的现代诗"、"编写一个Python函数计算斐波那契数列"）

## 2. 基于常识或专业知识可直接响应的查询
- 事实性问题（如"法国的首都是哪里"、"水的化学分子式是什么"、"2024年是闰年吗"）
- 概念解释请求（如"什么是区块链"、"解释一下机器学习的基本原理"、"TCP和UDP的区别"）
- 操作指导询问（如"如何在Excel中制作饼图"、"Python中如何读取CSV文件"）

## 3. 具有明确评判标准的分析请求
- 比较分析（如"iPhone和安卓手机的优缺点对比"、"Python和Java在数据处理方面的差异"）
- 优缺点评价（如"远程办公的利弊分析"、"新能源汽车值得购买吗"）
- 可行性评估（如"小型餐厅使用自助点餐系统是否合适"）

## 4. 基于上下文语境清晰的后续交互
- 确认类回应（如"好的"、"明白了"、"继续"）在有明确上下文时
- 修改完善请求（如"再详细一点"、"换个角度解释"、"用更简单的话说"）在已有讨论主题时
- 追问和深入（如"为什么会这样"、"还有其他方法吗"）在有具体讨论内容时

## 5. 表达明确情感或态度的交际性输入
- 感谢表达（如"谢谢你的帮助"、"这个解答很有用"）
- 情感分享（如"我对这个结果很满意"、"今天心情不错想聊聊天"）
- 礼貌性问候（如"你好"、"早上好"、"最近怎么样"）

# 判断示例

1. **给定完整题目但无解答要求** → 不属于模糊指令  
   *理由：一般情况下，给定题目就是用来解答的*

2. **仅给定选项（"A. B. C. D."）或仅给定问题干** → 属于模糊指令  
   *理由：仅凭选项或不完整问题无法作答*

3. **给定代码和报错信息但无debug要求** → 不属于模糊指令  
   *理由：一般情况下，提供报错信息大概率是要求调试*

4. **仅给定代码片段** → 大概率属于模糊指令  
   *理由：不确定是要执行、解释还是调试，除非代码有明显空缺暗示补全需求*

5. **"我想要买车，要求性能强"** → 不属于模糊指令  
   *理由：虽然用户未详述具体情况，但已有基本背景，可推测需要购车建议*

6. **"帮我写个代码，要求性能强"** → 属于模糊指令  
   *理由：虽然有指令，但"写代码"过于宽泛，缺少语言、功能等关键信息*

7. **"你想我吗？"** → 不属于模糊指令  
   *理由：这是对话场景的需求，常见于拟人化、角色扮演等场景*

8. **"你觉得呢？"** → 属于模糊指令  
   *理由：缺乏具体语境，不知道要对什么事物发表看法*

9. **"想象李白被贬的场景，写一个五言诗"** → 不属于模糊指令  
   *理由：有明确指令，李白是历史名人，其被贬经历可凭历史知识创作*

10. **"想象张美丽被贬的场景，写一个五言诗"** → 属于模糊指令  
    *理由：有明确指令，但不知道张美丽是谁，缺乏创作所需信息*

11. **"今天的日期"** → 不属于模糊指令  
    *理由：虽然简略，但可大概率推测用户要询问今天的日期*

12. **"今天的"** → 属于模糊指令  
    *理由：句子不完整，缺少关键信息*

13. **"给我用C语言写一个快排。sort(array, reverse) ..."** → 不属于模糊指令  
    *理由：首先提出明确要求，后续片段可理解为接口示例，整体意图清晰*

14. **"sort(array, reverse)"** → 属于模糊指令  
    *理由：仅有代码片段，无明确指令，不确定用户真实需求*
