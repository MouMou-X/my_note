## 什么是 Cursor

Cursor 是一个由 AI 驱动的代码编辑器，基于 VS Code 构建。它的核心特点是能够理解你的代码上下文，并提供智能的代码补全、编辑建议和问题解答。

### 主要优势
- 🤖 **AI 辅助编程**：利用 AI 理解代码上下文，提供智能建议
- 📝 **对话式编程**：可以直接与 AI 对话，让它帮你写代码
- 🔍 **上下文感知**：AI 能够理解整个项目结构，提供更准确的帮助
- ⚡ **快速迭代**：可以快速生成、修改和优化代码

## 核心功能

### 1. AI 对话（Chat）
- **位置**：右侧边栏或快捷键 `Cmd+L` (Mac) / `Ctrl+L` (Windows)
- **功能**：与 AI 对话，询问代码问题、请求代码生成、解释代码等
- **使用场景**：
  - "帮我写一个函数来计算斐波那契数列"
  - "解释这段代码的作用"
  - "如何优化这段代码？"

### 2. 内联编辑（Composer）
- **位置**：快捷键 `Cmd+I` (Mac) / `Ctrl+I` (Windows)
- **功能**：在代码中直接进行多文件编辑
- **使用场景**：
  - 重构代码
  - 添加新功能涉及多个文件
  - 修复 bug 需要修改多处

### 3. 自动补全（Tab 补全）
- **功能**：输入代码时，AI 会自动建议补全
- **使用**：按 `Tab` 键接受建议，`Esc` 键拒绝
- **设置**：可以在设置中调整补全的触发方式

### 4. 代码解释（Explain Code）
- **功能**：选中代码，AI 会解释其功能
- **使用**：选中代码后右键选择 "Explain" 或使用快捷键

## 基础操作

### 初次使用

1. **打开项目**
   - 使用 `File > Open Folder` 打开你的项目文件夹
   - Cursor 会自动识别项目类型和结构

2. **开始对话**
   - 按 `Cmd+L` 打开 AI 对话窗口
   - 输入你的问题或需求
   - 例如："帮我创建一个简单的待办事项应用"

3. **接受 AI 建议**
   - 当 AI 提供代码时，可以：
     - 直接接受（如果满意）
     - 要求修改（"请添加错误处理"）
     - 拒绝并重新请求

### 常用快捷键

| 快捷键 | 功能 |
|--------|------|
| `Cmd+L` / `Ctrl+L` | 打开 AI 对话窗口 |
| `Cmd+I` / `Ctrl+I` | 打开多文件编辑器（Composer） |
| `Tab` | 接受 AI 补全建议 |
| `Esc` | 拒绝 AI 建议 |
| `Cmd+K` / `Ctrl+K` | 内联编辑（在代码中直接编辑） |
| `Cmd+/` / `Ctrl+/` | 添加/删除注释 |

### 与 AI 对话的技巧

#### ✅ 好的提问方式
- **具体明确**："创建一个计算器类，支持加减乘除四种运算"
- **提供上下文**："在现有的 User 类中添加一个验证邮箱的方法"
- **分步骤**："先实现登录功能，然后再添加记住密码功能"

#### ❌ 避免的提问方式
- 过于模糊："帮我写代码"
- 缺少上下文："修复这个错误"（但没有说明是什么错误）
- 一次性要求太多："创建一个完整的电商网站"

## 最佳实践

### 1. 提供清晰的上下文
- 在提问时，说明你正在做什么
- 如果涉及多个文件，可以提到相关文件
- 提供错误信息或预期行为

### 2. 迭代式开发
- 不要一次性要求太多功能
- 先实现核心功能，再逐步添加特性
- 每完成一步，测试后再继续

### 3. 理解生成的代码
- 不要盲目接受所有建议
- 理解代码逻辑，确保它符合你的需求
- 学习 AI 生成的代码，提高自己的编程能力

### 4. 验证和测试
- AI 生成的代码可能不完美
- 总是测试代码是否按预期工作
- 检查边界情况和错误处理

### 5. 保持代码风格一致
- 告诉 AI 你偏好的代码风格
- 例如："使用 TypeScript，遵循 Airbnb 风格指南"

## 常见问题

### Q1: AI 生成的代码有错误怎么办？
**A**: 
- 将错误信息发送给 AI，让它修复
- 例如："这段代码报错了：TypeError: ...，请修复"

### Q2: 如何让 AI 理解我的项目结构？
**A**:
- Cursor 会自动读取项目文件
- 在对话中可以提到相关文件："查看 User.js 文件，然后..."
- 使用 Composer (`Cmd+I`) 进行多文件编辑时，AI 会自动理解文件关系

### Q3: 如何提高 AI 建议的质量？
**A**:
- 提供更详细的上下文
- 明确说明你的需求和约束
- 如果建议不满意，可以要求："请用另一种方式实现"

### Q4: Cursor 会发送我的代码到服务器吗？
**A**:
- Cursor 会发送代码到 AI 服务器进行处理
- 如果担心隐私，可以：
  - 使用本地模型（如果支持）
  - 不要在代码中包含敏感信息
  - 查看 Cursor 的隐私政策

### Q5: 如何撤销 AI 的更改？
**A**:
- 使用 `Cmd+Z` / `Ctrl+Z` 撤销
- 或者使用 Git 版本控制回退

### Q6: 可以自定义 AI 的行为吗？
**A**:
- 在设置中可以调整 AI 模型
- 可以在对话中明确指定："使用函数式编程风格"
- 创建 `.cursorrules` 文件来设置项目级别的规则
