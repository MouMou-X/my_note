

# **核心原则（方向对了，效率才爆表）**

- **问题驱动**：先列出“这个领域最经典的20个问题/任务”，再围绕它们学，而不是从目录学。
    
- **刻意练习**：永远在**舒适区外4–8%**的难度带；每次只打一个薄弱点。
    
- **快速闭环**：学—做—反馈—修正的回合越短越好（小时级，而非天/周）。
    
- **可迁移性**：学到的每条结论都要写下“在别处如何用？”（类比与反例）。
    
- **压缩→再展开**：先把一本书/一门课压缩成一页要点，再从要点向外展开细节。
    
- **第一性原理 + 玩具模型**：先用最简模型跑通直觉，再加现实约束校正。
    
- **写来思考**：把思考外化成图、表、推导和教别人用的说明书。
    

  

# **高效技巧（立刻可用）**

- **主动回忆 + 间隔重复**：读完就合上，用问题自测；把错题做成卡片复习。
    
- **交错练习**：混练相似但不同的题型，逼迫你识别“何时用哪招”。
    
- **费曼技巧**：用高中生能懂的话写一页讲解，暴露知识漏洞。
    
- **错误账本**：记录“错在哪里—正确做法—触发提醒语”；每周复盘Top 5错误。
    
- **双编码**：同一概念用图示/流程图/公式/口语四种方式表达一次。
    
- **极端检验**：把参数拉到极端，看看结论是否仍成立，逼出边界条件。
    

  

# **学习流水线（可复用模板）**

1. **选题**：定义1个“单位能力”（例如：能在30分钟内从零复现X算法/证明Y结论）。
    
2. **拆解**：列出达成它需要的3–5个微技能。
    
3. **资料**：只挑2个主资源 + 1个参考，不“囤书”。
    
4. **训练集**：准备10–20道“典型任务”，按难度梯度排序。
    
5. **回路**：每次训练=定时30–90分钟深度块 → 当场自测 → 对标“金标准答案”。
    
6. **产出**：输出一页讲义、1段代码、或1个小实验复现。
    
7. **度量**：记录用时、正确率、是否能**闭卷**完成、是否能**迁移**到陌生题。
    

  

# **用AI放大（把AI当“苏格拉底导师 + 严苛评审”）**

- 让AI**只问不讲**：请它按“由浅入深的提问树”盘问你，直到你讲清每一步因果。
    
- 让AI**出题与改题**：基于你的一页讲义生成自测题，要求给出**错因分析**。
    
- 让AI**反对你**：让它从相反立场找出漏洞，训练你的“红队化”思维。
    
- 让AI**压缩与展开**：把长文压成“关键式—机制—应用—陷阱”四格，再让它帮你从四格展开例子与反例。
    

  

# **一周上手计划（每天≤2小时深度块）**

- **Day 1 领域地图**：列20个典型问题；写“为什么这些是核心”。
    
- **Day 2 玩具模型**：用最小例子跑通核心机制，并画出流程图。
    
- **Day 3 金标准复现**：找1个权威解/论文/项目，闭卷复现到能跑通。
    
- **Day 4 闭卷实操**：限时完成3道典型题；建立错误账本。
    
- **Day 5 教别人**：写一页讲义 + 5题自测；让AI当学生质询你。
    
- **Day 6 迁移练习**：把方法移到相近但不同的场景；做对比表。
    
- **Day 7 复盘与卡片化**：筛选一周Top 20知识点做成卡片，安排间隔复习。
    

  

# **低成本高回报的微习惯**

- 开场5分钟写下“今日唯一产出物是什么？”（一页讲义/一段代码/一张图）
    
- 结束5分钟写“下一步最小可行动作”（≤15分钟能完成）。
    
- 每天一个**极简复现**：把昨日结论用不看资料的方式重做一遍。
    
- 90分钟深度块内**零通知**，只留一个定时器。
    

  

# **常见坑**

- 只看不做、收藏资源当学习；
    
- 过度求全而没有“单位能力”的产出；
    
- 永远在会的题上刷存在感；
    
- 没有闭卷环节与计时，导致“会而不会”。
